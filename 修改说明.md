# 修改说明

## 问题描述
当在疾病管理页面点击检查牲畜健康，生成对应的疾病记录时，告警管理页面点击扫描告警，对应的牲畜健康告警没有显示在告警页面。

## 解决方案

### 修改内容
1. **在 DiseaseRecordService 类中注入 WarningService**
   - 导入 WarningService 类
   - 添加 WarningService 字段并使用 @Autowired 注解注入

2. **修改 addDiseaseRecord 方法**
   - 在添加疾病记录时，生成牲畜健康告警
   - 使用 warningService.generateWarning 方法创建告警记录

3. **修改 recoverDiseaseRecord 方法**
   - 在标记疾病康复时，消除牲畜健康告警
   - 使用 warningService.eliminateLivestockWarning 方法消除告警记录

### 技术细节
- 当添加疾病记录时，系统会自动生成一个类型为"牲畜"、级别为"高"的告警记录
- 当标记疾病康复时，系统会自动消除对应的告警记录
- 告警记录的来源为牲畜编号，详情为疾病名称

### 测试方法
1. 启动后端服务
2. 在疾病管理页面添加一条疾病记录
3. 打开告警管理页面，点击"扫描告警"按钮
4. 查看告警列表，应该能看到对应的牲畜健康告警
5. 在疾病管理页面标记该疾病记录为"已康复"
6. 再次点击"扫描告警"按钮
7. 查看告警列表，该告警应该已经被消除

## 预期效果
- 当在疾病管理页面添加疾病记录时，告警管理页面会同步增加告警记录
- 当在疾病管理页面标记疾病康复时，告警管理页面的对应告警会被消除
- 当点击告警管理页面的"扫描告警"按钮时，系统会检查所有牲畜的健康状态，并生成或消除对应的告警记录
